# How to estimate fff?

ggg is the distribution of data that is unknown

  1. Choose a model f_
    • Parametric: Assumes a specific form for fff, estimating a fixed set of parameters by fitting or training.
    • Non-parametric: Does not assume a specific form, but need a larger amount of data.
  2. Choose a quality measure
    • Loss function: measures how well a model fits the data.
    • Common loss functions:
      • Mean Squared Error (MSE): for regression problems

        MSE=1n∑i=1n(yi−y^i)2MSE = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2 MSE=n1​i=1∑n​(yi​−y^​i​)2

      • Cross-Entropy Loss: for classification problems

        L=−∑i=1n∑c=1Cyi,clog⁡(y^i,c)L = -\sum_{i=1}^{n} \sum_{c=1}^{C} y_{i,c} \log(\hat{y}_{i,c}) L=−i=1∑n​c=1∑C​yi,c​log(y^​i,c​)

  3. Optimize(fitting)
    • Find the best parameters θ\thetaθ
    • Calculus to find close form solution, gradient descent, expectation-maximization(EM) algorithm, etc.

# Nearest Neighbor Averaging

If we have few data points, we can use the average of nearest neighbors to estimate fff.
We cannot compute E(Y∣X=x)E(Y|X=x)E(Y∣X=x), but we can let f^(x)=Ave(Y∣X∈N(x))\hat{f}(x) = Ave(Y|X \in N(x))f^​(x)=Ave(Y∣X∈N(x)), where N(x)N(x)N(x) is some neighborhood of xxx.

Curse of dimensionality (維數災難): As the number of features increases, the volume of the feature space increases exponentially, making data points sparse. This sparsity makes it difficult to find nearby neighbors, reducing the effectiveness of nearest neighbor methods.

# Parametric and structured models

The linear model is an important example of a parametric model:

Y=β0+β1X1+β2X2+...+βpXp+ϵY = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + ... + \beta_p X_p + \epsilon Y=β0​+β1​X1​+β2​X2​+...+βp​Xp​+ϵ

  • Specified in terms of p+1p+1p+1 parameters: β0,β1,...,βp\beta_0, \beta_1, ..., \beta_pβ0​,β1​,...,βp​
  • We estimate the parameters by fitting the model to the training data.
  • It often serves as a good and interpretable approximation to the true function f(X)f(X)f(X).

# Some tradeoffs

  • Prediction accuracy vs. interpretability
    Linear models are more interpretable; thin-plate splines are not
  • Good fit vs. overfit
  • Parsimony vs. black box
    Prefer a simpler model over a black-box predictor

# Bias-Variance Tradeoff

Suppose we have fit model f^\hat{f}f^​ to some training data, and let (x0,y0)(x_0, y_0)(x0​,y0​) be a test observation drawn from the population.
The expected test MSE at a point x0x_0x0​ can be decomposed into three fundamental components:

E[(y0−f^(x0))2]=BiasTr[f^(x0,Tr)]2+VarTr[f^(x0,Tr)]+Var(ϵ)E[(y_0 - \hat{f}(x_0))^2] = Bias_{Tr}[\hat{f}(x_0,Tr)]^2 + Var_{Tr}[\hat{f}(x_0,Tr)] + Var(\epsilon) E[(y0​−f^​(x0​))2]=BiasTr​[f^​(x0​,Tr)]2+VarTr​[f^​(x0​,Tr)]+Var(ϵ)
